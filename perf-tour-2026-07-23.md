## Quick wins prioritarios (mañana temprano)

1. **Índices `fact_ventas_unisuper`** — impacta 6 de las 10 queries top.
   ```sql
   CREATE INDEX IF NOT EXISTS idx_fvu_pais_fecha    ON fact_ventas_unisuper (pais, fecha);
   CREATE INDEX IF NOT EXISTS idx_fvu_pais_cadena   ON fact_ventas_unisuper (pais, cadena, fecha);
   CREATE INDEX IF NOT EXISTS idx_fvu_subcat        ON fact_ventas_unisuper (pais, subcategoria);
   ```
   Alternativa mejor: crear `mv_unisuper_mensual` (paralelo a `mv_exito_mensual`, `mv_walmart_mensual`).

2. **`v_ventas` → materializar como `mv_ventas_90d`** — 5.8s
   La vista corre `MAKE_DATE(ano::int, mes::int, dia::int)` en cada fila. Como MV con índice sobre `fecha` es < 100ms.

3. **`fact_ventas_selectos` índices** — para insights (varios de ~1-1.5s).
   ```sql
   CREATE INDEX IF NOT EXISTS idx_fvs_pais_fecha  ON fact_ventas_selectos (pais, fecha);
   CREATE INDEX IF NOT EXISTS idx_fvs_barra_fecha ON fact_ventas_selectos (codigo_barras, fecha);
   ```

4. **`fact_selectos_inventario` índices** — 3 queries en 1300ms cada una.
   ```sql
   CREATE INDEX IF NOT EXISTS idx_fsi_fecha_barra ON fact_selectos_inventario (fecha, codigo_barra);
   CREATE INDEX IF NOT EXISTS idx_fsi_fecha_tienda ON fact_selectos_inventario (fecha, tienda);
   ```

5. **`mv_exito_mensual` — verificar índices y refresh** — 9.6s en WITH cur.
   ```sql
   SELECT * FROM pg_indexes WHERE tablename = 'mv_exito_mensual';
   -- si no hay:
   CREATE INDEX IF NOT EXISTS idx_mv_exito_pais_ano_mes ON mv_exito_mensual (pais, ano, mes);
   ```

6. **NSE inline VALUES en cobertura** — el nse_map con 40+ códigos hardcodeados se manda en cada request. Migrar a tabla `dim_selectos_pdv (codigo, nse)` y hacer JOIN — reduce query size + permite caching.

7. **`/co/exito/kpis` cold start 10s** — al segundo hit debería bajar por prepared statements + cache HTTP. Verificar `revalidate` en `route.ts`.

8. **`/co/exito/seguimiento` 6.7s** — considerar `mv_exito_seguimiento` (una MV que ya tenga los agregados por producto/cadena/subformato/departamento con RR y proyección precomputados).
